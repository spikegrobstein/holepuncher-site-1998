Since it is an editor for interface altering skins, HolePuncher must contain tools for easily managing all resources and items that you would need to edit such a thing. For those of you familiar with REALbasic, HolePuncher's user interface will be very familiar in that it is primarily based on that.

As part of the HolePuncher interface, the project window is where you will find a list of major components (ie pictures, sounds, and windows). By double clicking on an item, you can play the sound, preview the picture, or open a window for editing. Once a window is open, you can move components around either with the mouse, arrow keys, or numericly using the properties window. The properties window not only allows you to edit the position and size of controls, but also is the place where you will edit the caption that appears on buttons, the title of windows, and many more things.

The Tools window is where you create new controls in the window. Simply by clicking and dragging, it picks up a control and is ready to be dropped into a window.

HolePuncher also has full contextual menu support that allow you to do just about anything you would want to with the object that you brought the menu up from. Simply by control-clicking on a control, you can cut, copy, add an action, remove all actions, of delete the control, or if you have an action on the clipboard, you can paste an action into it.

Actions allow you to give buttons and other controls functionality in a TinHole skin. All you have to do is define what triggers the action (mouse click for example) and type in some of the easy to learn scripting language, and you have added an action. Skins also take full advantage of TinHole's plugins and helper applications. If you write a plugin for Tinhole's use, you can reference commands of it from your skin's actions.

These are the controls that HolePuncher currently supports:

    PushButton- A standard button. Supports icon and/or text caption
    PictureBox- Use it to place a picture in a window; does not support actions.
    Placard- Used to fill space, take advantage of TinHole's color scheme, or have live window drag.
    StaticText- Place text inside your window with color and font information.
    EditField- Standard text input box. Supports color and font customization.
    DisclosureTriangle- A triangle that can be used to resize controls or windows by clicking it.
    NubBox- A beveled box with an optional nub. Color is customizable.
    PopupMenu- Add a popup menu to your skin.
    ListBox- Used to add a user or file list to your skin.
    GroupBox- To visually group controls.
